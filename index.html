Architected for resilience.

Designed to strict specifications with isolated responsibilities, robust healthchecks, and data persistence guarantees.

Microservices

Decoupled frontend proxy, Python API application, and database. Managed holistically via Docker Compose.

LAN Static Routing

Direct IP address assignment to containers via `macvlan`, fully bypassing host network address translation (NAT).

Persistent Volumes

PostgreSQL data is mounted via Docker named volumes, ensuring analysis records survive container lifecycles.

Advanced Topologies

This environment demonstrates complex Docker networking bridging the gap between host and LAN.

  • Requires external macvlan network creation
  • IP collision mitigation via explicit subnet masks
  • Dual-homed backend handles both LAN & local proxy
docker-compose.yml
services: backend: container_name: image-analyzer-api depends_on: database: condition: service_healthy # Connect to both internal bridge and external LAN networks: app_net: {} lan_net: ipv4_address: ${BACKEND_STATIC_IP} # Host-isolation workaround for local testing frontend_local: ports: - "${FRONTEND_HOST_PORT:-8088}:80" networks: app_net: {} networks: lan_net: external: true name: ${DOCKER_LAN_NETWORK}
