Docker Orchestration

Multi-container setup via Docker Compose with explicit startup dependencies, healthchecks, and robust container lifecycle management.

Macvlan & Ipvlan

Containers join external networks as first-class LAN endpoints with static IPs, demonstrating advanced real-world Docker networking concepts.

High Resilience

Engineered with robust unless-stopped policies, dual-homed networks for local testing, and seamless host-isolation workarounds.

Advanced LAN Deployment

Unlike standard bridge networks, this project utilizes macvlan or ipvlan drivers to attach containers directly to the physical network seamlessly.

Macvlan Mode (Recommended)

Assigns a unique MAC address to each container. Containers appear as separate physical devices on the router, excellent for vivid LAN demonstrations.

Localhost Workaround

To solve macvlan host-isolation (where a host cannot talk to its own macvlan containers), a dedicated frontend_local service runs on a bridge network, exposed on localhost.

Dual-Homed Architecture

The FastAPI backend connects to both the external LAN network and the internal bridge, seamlessly handling external requests alongside local testing routes.

docker-compose.yml

services:
  backend:
    # Dual-homed network attachment
    networks:
      app_net: {}
      lan_net:
        ipv4_address: ${BACKEND_STATIC_IP}

networks:
  app_net:
    driver: bridge
  lan_net:
    # Represents macvlan/ipvlan created externally
    external: true
    name: ${DOCKER_LAN_NETWORK}
